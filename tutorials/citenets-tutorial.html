citenets: Citation and Bibliometric Network Construction in R

A Comprehensive Tutorial with Equivalence Testing

Author

Mohammed Saqr

Published

April 15, 2026

1. Why citenets?

Bibliometric network analysis requires constructing networks from scholarly data — who collaborates with whom, which papers are cited together, what keywords co-occur. Existing tools either bundle everything into a monolithic package with 87+ dependencies (bibliometrix) or cover only fragments of the problem (biblionetwork).

citenets fills the gap: network construction only, with state-of-the-art counting and normalization methods, minimal dependencies, and an intuitive API.

Feature citenets bibliometrix biblionetwork
Counting methods 13 1 (full) 3
Similarity measures 6 5 1 (cosine)
Network types 8 functions 4 3
Dependencies 3 87+ 3
Output Data frame Sparse matrix Data table 
devtools::load_all("..")

2. Reading Bibliometric Data

citenets reads data from six sources into a unified tidy format: every reader returns the same column structure with human-readable names and list-columns for multi-valued fields.

3. Network Types

citenets builds seven types of entity networks plus a generic co-occurrence function.

4. Counting Methods — The Core Innovation

Counting methods determine how credit is distributed among co-authors or how co-occurrences are weighted. citenets implements 13 methods — 9 more than any other R package.

4.1 Position-Independent Methods (4)

These treat all authors equally regardless of byline position.

d <- data.frame(id = "W1", stringsAsFactors = FALSE)
d$authors <- list(c("Alice", "Bob", "Carol"))

# Full counting: each link = 1
author_network(d, "collaboration", counting = "full")
   from    to weight count shared
1 Alice   Bob      1     1      1
2 Alice Carol      1     1      1
3   Bob Carol      1     1      1
# Fractional (Perianes-Rodriguez): each author's total = 1
# Link weight = 1/(n-1) = 0.5
author_network(d, "collaboration", counting = "fractional")
   from    to weight count shared
1 Alice   Bob    0.5     1      1
2 Alice Carol    0.5     1      1
3   Bob Carol    0.5     1      1
# Paper-level fractional (Batagelj): paper total = 1
# Link weight = 2/(n*(n-1)) = 1/3
author_network(d, "collaboration", counting = "paper")
   from    to    weight count shared
1 Alice   Bob 0.3333333     1      1
2 Alice Carol 0.3333333     1      1
3   Bob Carol 0.3333333     1      1

The difference: fractional normalizes per author (each author’s total co-authorship from this paper = 1). Paper normalizes per paper (the paper’s total contribution to the network = 1).

# Verify Batagelj property: total contribution per paper = 1
edges <- author_network(d, "collaboration", counting = "paper")
sum(edges$weight)  # Should be 1.0
[1] 1

Harmonic (Hagen 2008)

Credit proportional to 1/position. The first author always gets twice the credit of the second, the second 1.5 times the third, etc.

Geometric

Exponential decay: 0.5^(position-1). Concentrates credit heavily on early authors.

Adaptive Geometric (Liu & Fang 2023)

The ratio of first to last author credit equals the number of authors. Adapts to team size.

# Verify: first/last ratio = n
for (n in c(3, 5, 10)) {
  w <- author_weights(n, "adaptive_geometric")
  cat(sprintf("n=%2d: first/last = %.2f (expected %d)\n", n, w[1]/w[n], n))
}
n= 3: first/last = 3.00 (expected 3)
n= 5: first/last = 5.00 (expected 5)
n=10: first/last = 10.00 (expected 10)

Golden Ratio

Uses phi (1.618…) as the decay constant. Consecutive authors have a ratio of phi.

4.3 How Position-Dependent Counting Affects Network Edges

The key mechanism: author positional weights are placed directly into the bipartite matrix. Matrix multiplication then gives edge weight = w_i * w_j per shared paper. Higher-ranked author pairs get stronger edges.

# One paper: Alice (pos 1), Bob (pos 2), Carol (pos 3)
d <- data.frame(id = "W1", stringsAsFactors = FALSE)
d$authors <- list(c("Alice", "Bob", "Carol"))

# Harmonic weights: Alice = 6/11, Bob = 3/11, Carol = 2/11
w <- author_weights(3, "harmonic")
cat("Weights:", round(w, 4), "\n")
Weights: 0.5455 0.2727 0.1818 
# Edge weight = product of positional weights
edges <- author_network(d, "collaboration", counting = "harmonic")
cat("Alice-Bob:", round(w[1] * w[2], 4), "=", round(edges$weight[edges$from == "Alice" & edges$to == "Bob"], 4), "\n")
Alice-Bob: 0.1488 = 0.1488 
cat("Alice-Carol:", round(w[1] * w[3], 4), "=", round(edges$weight[edges$from == "Alice" & edges$to == "Carol"], 4), "\n")
Alice-Carol: 0.0992 = 0.0992 
cat("Bob-Carol:", round(w[2] * w[3], 4), "=", round(edges$weight[edges$from == "Bob" & edges$to == "Carol"], 4), "\n")
Bob-Carol: 0.0496 = 0.0496

5. Similarity Measures

After counting co-occurrences, similarity measures normalize edge weights to account for how active each entity is. An entity that appears in 100 papers will naturally co-occur with more entities — normalization corrects for this.

5.1 The Six Measures

Given co-occurrence count c_ij and total occurrences w_i, w_j:

Similarity Formula When to use
"none" c_ij Raw counts, no correction
"association" c_ij / (w_i * w_j) Recommended default (van Eck & Waltman 2009)
"cosine" c_ij / sqrt(w_i * w_j) Coupling angle (Sen & Gan 1983)
"jaccard" c_ij / (w_i + w_j - c_ij) Set overlap
"inclusion" c_ij / min(w_i, w_j) Subset detection
"equivalence" c_ij^2 / (w_i * w_j) Cosine squared 
# Same network, different similarity measures
methods <- c("none", "association", "cosine", "jaccard", "inclusion", "equivalence")

results <- lapply(methods, function(m) {
  e <- reference_network(biblio_data, similarity = m, threshold = 0)
  data.frame(similarity = m,
             edges = nrow(e),
             min_weight = round(min(e$weight), 4),
             max_weight = round(max(e$weight), 4),
             mean_weight = round(mean(e$weight), 4))
})
do.call(rbind, results)
   similarity edges min_weight max_weight mean_weight
1        none    38     1.0000          4      1.5789
2 association    38     0.0476          1      0.1811
3      cosine    38     0.2182          1      0.4822
4     jaccard    38     0.1111          1      0.3180

5.2 Association Strength — The Recommended Default

Association strength measures how much more often two entities co-occur than expected by chance. A value of 1 means exactly as expected; >1 means they co-occur more than random; <1 means less.

edges <- reference_network(biblio_data, similarity = "association", threshold = 0)
head(edges)
  from to weight count shared
1   R9 W5   1.00     1      1
2   R7 R9   0.50     1      1
3   R7 W5   0.50     1      1
4  R10 R6   0.25     1      1
5   R6 W2   0.25     2      2
6  R10 W3   0.25     1      1

8. Equivalence Testing — citenets vs bibliometrix vs biblionetwork

The following sections verify that citenets produces numerically identical results to established packages.

8.1 Co-authorship: citenets vs bibliometrix

11. Quick Reference

All Counting Methods

Method Type Formula (author i of n)
"full" Position-independent 1 for all
"fractional" Position-independent 1/(n-1) per link
"paper" Position-independent 2/(n*(n-1)) per link
"strength" Position-independent IDF-weighted
"harmonic" Position-dependent (1/i) / H_n
"arithmetic" Position-dependent (n-i+1) / sum(1:n)
"geometric" Position-dependent 0.5^(i-1) normalized
"adaptive_geometric" Position-dependent r^(n-i), r=n^(1/(n-1))
"golden" Position-dependent phi^(n-i) normalized
"first" Special First author = 1, rest = 0
"last" Special Last author = 1, rest = 0
"first_last" Special First + last elevated
"position_weighted" Custom User-defined vector

All Similarity Measures

Similarity Formula
"none" Raw co-occurrence count
"association" c_ij / (w_i * w_j)
"cosine" c_ij / sqrt(w_i * w_j)
"jaccard" c_ij / (w_i + w_j - c_ij)
"inclusion" c_ij / min(w_i, w_j)
"equivalence" c_ij^2 / (w_i * w_j)

## 1. Why citenets?

Bibliometric network analysis requires constructing networks from scholarly data — who collaborates with whom, which papers are cited together, what keywords co-occur. Existing tools either bundle everything into a monolithic package with 87+ dependencies (bibliometrix) or cover only fragments of the problem (biblionetwork).

citenets fills the gap: **network construction only**, with state-of-the-art counting and normalization methods, minimal dependencies, and an intuitive API.

| Feature | citenets | bibliometrix | biblionetwork |
|---------|----------|--------------|---------------|
| **Counting methods** | 13 | 1 (full) | 3 |
| **Similarity measures** | 6 | 5 | 1 (cosine) |
| **Network types** | 8 functions | 4 | 3 |
| **Dependencies** | 3 | 87+ | 3 |
| **Output** | Data frame | Sparse matrix | Data table |

## 3. Network Types

citenets builds seven types of entity networks plus a generic co-occurrence function.

### 3.1 Author Collaboration Network

Two authors are linked when they co-author a paper. The most common bibliometric network.

```{r}
#| label: author-collab
edges <- author_network(biblio_data, "collaboration")
edges
```

Each row is an edge. `weight` = co-authorship count, `count` = raw count, `shared` = number of shared papers.

### 3.2 Reference Co-citation Network

Two references are linked when they are cited together by the same paper. Co-citation strength reflects intellectual structure — which foundational works are perceived as related.

```{r}
#| label: ref-cocit
edges <- reference_network(biblio_data, threshold = 0)
head(edges, 10)
```

### 3.3 Document Bibliographic Coupling

Two documents are linked when they share cited references. Unlike co-citation (retrospective), coupling is prospective — computable immediately upon publication.

```{r}
#| label: doc-coupling
edges <- document_network(biblio_data, "coupling")
head(edges)
```

### 3.4 Direct Citation Network

Directed edges from citing to cited documents. Only internal citations (both documents present in the dataset) are included.

```{r}
#| label: direct-cit
edges <- document_network(biblio_data, "citation")
edges
```

### 3.5 Keyword Co-occurrence Network

Two keywords are linked when they appear in the same paper.

```{r}
#| label: keyword-cooc
edges <- keyword_network(biblio_data, threshold = 0)
head(edges, 10)
```

### 3.6 Other Entity Networks

Institution, country, and source (journal) networks follow the same pattern:

```r
# Institutions linked by co-authorship
institution_network(data, "collaboration")

# Countries linked by shared references
country_network(data, "coupling")

# Journals linked by co-citation
source_network(data, "co_citation")
```

Each supports types: `"collaboration"`, `"coupling"`, `"co_citation"`, `"equivalence"` (where applicable).

## 4. Counting Methods — The Core Innovation

Counting methods determine how credit is distributed among co-authors or how co-occurrences are weighted. citenets implements 13 methods — 9 more than any other R package.

### 4.1 Position-Independent Methods (4)

These treat all authors equally regardless of byline position.

```{r}
#| label: counting-independent
d <- data.frame(id = "W1", stringsAsFactors = FALSE)
d$authors <- list(c("Alice", "Bob", "Carol"))

# Full counting: each link = 1
author_network(d, "collaboration", counting = "full")

# Fractional (Perianes-Rodriguez): each author's total = 1
# Link weight = 1/(n-1) = 0.5
author_network(d, "collaboration", counting = "fractional")

# Paper-level fractional (Batagelj): paper total = 1
# Link weight = 2/(n*(n-1)) = 1/3
author_network(d, "collaboration", counting = "paper")
```

The difference: **fractional** normalizes per author (each author's total co-authorship from this paper = 1). **Paper** normalizes per paper (the paper's total contribution to the network = 1).

```{r}
#| label: counting-paper-verify
# Verify Batagelj property: total contribution per paper = 1
edges <- author_network(d, "collaboration", counting = "paper")
sum(edges$weight)  # Should be 1.0
```

### 4.2 Position-Dependent Methods (9)

These respect author byline order — first author gets more credit.

```{r}
#| label: counting-weights
# Compare all position-dependent methods for a 5-author paper
methods <- c("harmonic", "arithmetic", "geometric",
             "adaptive_geometric", "golden",
             "first", "last", "first_last")

weights_table <- do.call(rbind, lapply(methods, function(m) {
  w <- author_weights(5, m)
  data.frame(method = m, pos1 = w[1], pos2 = w[2], pos3 = w[3],
             pos4 = w[4], pos5 = w[5])
}))
weights_table
```

#### Harmonic (Hagen 2008)

Credit proportional to 1/position. The first author always gets twice the credit of the second, the second 1.5 times the third, etc.

```{r}
#| label: harmonic
author_network(biblio_data, "collaboration", counting = "harmonic") |>
  head(5)
```

#### Geometric

Exponential decay: 0.5^(position-1). Concentrates credit heavily on early authors.

```{r}
#| label: geometric
author_network(biblio_data, "collaboration", counting = "geometric") |>
  head(5)
```

#### Adaptive Geometric (Liu & Fang 2023)

The ratio of first to last author credit equals the number of authors. Adapts to team size.

```{r}
#| label: adaptive
# Verify: first/last ratio = n
for (n in c(3, 5, 10)) {
  w <- author_weights(n, "adaptive_geometric")
  cat(sprintf("n=%2d: first/last = %.2f (expected %d)\n", n, w[1]/w[n], n))
}
```

#### Golden Ratio

Uses phi (1.618...) as the decay constant. Consecutive authors have a ratio of phi.

```{r}
#| label: golden
w <- author_weights(5, "golden")
phi <- (1 + sqrt(5)) / 2
cat("Consecutive ratios:", paste(round(w[1:4] / w[2:5], 3), collapse = ", "), "\n")
cat("Expected phi:", round(phi, 3), "\n")
```

#### First / Last / First-Last

```{r}
#| label: first-last
# First author only: others get zero weight → no co-authorship edges
author_weights(4, "first")

# Last author only
author_weights(4, "last")

# First and last elevated, middle share rest
author_weights(4, "first_last")
```

#### Position Weighted (custom)

User-defined weight vector:

```{r}
#| label: position-weighted
# Custom: heavy first, moderate second, equal rest
author_network(biblio_data, "collaboration",
               counting = "position_weighted",
               position_weights = c(1.0, 0.7, 0.3))  |>
  head(5)
```

### 4.3 How Position-Dependent Counting Affects Network Edges

The key mechanism: author positional weights are placed directly into the bipartite matrix. Matrix multiplication then gives edge weight = w_i * w_j per shared paper. Higher-ranked author pairs get stronger edges.

```{r}
#| label: counting-mechanism
# One paper: Alice (pos 1), Bob (pos 2), Carol (pos 3)
d <- data.frame(id = "W1", stringsAsFactors = FALSE)
d$authors <- list(c("Alice", "Bob", "Carol"))

# Harmonic weights: Alice = 6/11, Bob = 3/11, Carol = 2/11
w <- author_weights(3, "harmonic")
cat("Weights:", round(w, 4), "\n")

# Edge weight = product of positional weights
edges <- author_network(d, "collaboration", counting = "harmonic")
cat("Alice-Bob:", round(w[1] * w[2], 4), "=", round(edges$weight[edges$from == "Alice" & edges$to == "Bob"], 4), "\n")
cat("Alice-Carol:", round(w[1] * w[3], 4), "=", round(edges$weight[edges$from == "Alice" & edges$to == "Carol"], 4), "\n")
cat("Bob-Carol:", round(w[2] * w[3], 4), "=", round(edges$weight[edges$from == "Bob" & edges$to == "Carol"], 4), "\n")
```

## 5. Similarity Measures

After counting co-occurrences, similarity measures normalize edge weights to account for how active each entity is. An entity that appears in 100 papers will naturally co-occur with more entities — normalization corrects for this.

### 5.1 The Six Measures

Given co-occurrence count c_ij and total occurrences w_i, w_j:

| Similarity | Formula | When to use |
|------------|---------|-------------|
| `"none"` | c_ij | Raw counts, no correction |
| `"association"` | c_ij / (w_i * w_j) | Recommended default (van Eck & Waltman 2009) |
| `"cosine"` | c_ij / sqrt(w_i * w_j) | Coupling angle (Sen & Gan 1983) |
| `"jaccard"` | c_ij / (w_i + w_j - c_ij) | Set overlap |
| `"inclusion"` | c_ij / min(w_i, w_j) | Subset detection |
| `"equivalence"` | c_ij^2 / (w_i * w_j) | Cosine squared |

```{r}
#| label: similarity-compare
# Same network, different similarity measures
methods <- c("none", "association", "cosine", "jaccard", "inclusion", "equivalence")

results <- lapply(methods, function(m) {
  e <- reference_network(biblio_data, similarity = m, threshold = 0)
  data.frame(similarity = m,
             edges = nrow(e),
             min_weight = round(min(e$weight), 4),
             max_weight = round(max(e$weight), 4),
             mean_weight = round(mean(e$weight), 4))
})
do.call(rbind, results)
```

### 5.2 Association Strength — The Recommended Default

Association strength measures how much more often two entities co-occur than expected by chance. A value of 1 means exactly as expected; >1 means they co-occur more than random; <1 means less.

```{r}
#| label: association
edges <- reference_network(biblio_data, similarity = "association", threshold = 0)
head(edges)
```

### 5.3 Cosine Similarity = Coupling Angle

For bibliographic coupling, cosine normalization is equivalent to the "coupling angle" of Sen & Gan (1983).

```{r}
#| label: cosine-coupling
# Document coupling with cosine = coupling angle
edges <- document_network(biblio_data, "coupling", similarity = "cosine")
head(edges)
```

### 8.6 All Five Normalization Formulas: citenets vs bibliometrix

```{r}
#| label: equiv-normalize
# Build a known co-occurrence matrix
A <- matrix(
  c(4,2,2,1, 2,3,2,2, 2,2,3,1, 1,2,1,2),
  nrow = 4, dimnames = list(paste0("R",1:4), paste0("R",1:4))
)

methods_map <- list(
  association = "association",
  cosine = "salton",
  jaccard = "jaccard",
  inclusion = "inclusion",
  equivalence = "equivalence"
)

results <- lapply(names(methods_map), function(m) {
  cn <- as.matrix(normalize(A, m))
  bm <- as.matrix(bibliometrix::normalizeSimilarity(
    Matrix::Matrix(A, sparse = TRUE), type = methods_map[[m]]
  ))
  bm[is.nan(bm)] <- 0; diag(bm) <- 0
  max_diff <- max(abs(cn - bm))
  data.frame(method = m, max_diff = max_diff,
             match = ifelse(max_diff < 1e-10, "EXACT", "DIFFERS"))
})
do.call(rbind, results)
```

## 11. Quick Reference

### All Counting Methods

| Method | Type | Formula (author i of n) |
|--------|------|------------------------|
| `"full"` | Position-independent | 1 for all |
| `"fractional"` | Position-independent | 1/(n-1) per link |
| `"paper"` | Position-independent | 2/(n*(n-1)) per link |
| `"strength"` | Position-independent | IDF-weighted |
| `"harmonic"` | Position-dependent | (1/i) / H_n |
| `"arithmetic"` | Position-dependent | (n-i+1) / sum(1:n) |
| `"geometric"` | Position-dependent | 0.5^(i-1) normalized |
| `"adaptive_geometric"` | Position-dependent | r^(n-i), r=n^(1/(n-1)) |
| `"golden"` | Position-dependent | phi^(n-i) normalized |
| `"first"` | Special | First author = 1, rest = 0 |
| `"last"` | Special | Last author = 1, rest = 0 |
| `"first_last"` | Special | First + last elevated |
| `"position_weighted"` | Custom | User-defined vector |

### All Similarity Measures

| Similarity | Formula |
|------------|---------|
| `"none"` | Raw co-occurrence count |
| `"association"` | c_ij / (w_i * w_j) |
| `"cosine"` | c_ij / sqrt(w_i * w_j) |
| `"jaccard"` | c_ij / (w_i + w_j - c_ij) |
| `"inclusion"` | c_ij / min(w_i, w_j) |
| `"equivalence"` | c_ij^2 / (w_i * w_j) |
